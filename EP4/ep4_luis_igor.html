Training and evaluating models section

To select the best hyperparameters, we decided to evaluate every performance metric calculated. Therefore, we only updated the best hyperparameter values when we had a better result in all the metrics we used comparing to the last best case.

With respect to the confusion matrix, we decided to plot it to visually compare the model performance in each class of the data set so we could see if there is any case that stands out from the others in a bad or a good way.

Lastly, we plotted one graph for each hyperparameter that we tuned to better understand and visualize the influence of each of them to the model accuracy. To do this, we fixed the best values of all the other hyperparameters that were not beeing analyzed and line plotted all the tested values of the hyperparameter we wanted to study.

#Initializing the variables that will store the best hyperparameters values and also the best performance metrics values

best_f1 = 0

logistic_regression_best_clf = None

arg_iter = -1
arg_C = -1
accuracy_hist = []
f1_score_hist = []
log_loss_hist = []

#Fitting the model instances and selecting hyperparameters
cont_i = 0
cont_j = 0
for i in np.linspace(100, 1000, 10):
    for j in range(1, 11):
        logistic_clf = LogisticRegression(random_state=10, max_iter=i,
                                        tol=0.0001, C=j).fit(X_train_train, y_train_train)
        
        predictions = logistic_clf.predict(X_train_val)
        predictions_prob = logistic_clf.predict_proba(X_train_val)

        accuracy_val = logistic_clf.score(X_train_val, y_train_val)
        f1_score_val = f1_score(y_true=y_train_val, y_pred=predictions, labels=np.unique(y_train_val), average='macro')
        log_loss_val = log_loss(y_true=y_train_val, y_pred=predictions_prob, labels=np.unique(y_train_val))

        accuracy_hist.append((i, j, accuracy_val))
        f1_score_hist.append((i, j, f1_score_val))
        log_loss_hist.append((i, j, log_loss_val))
        
        #Selecting the best hyperparameters based on the performance metrics calculated on the sub-validation set.
        if accuracy_val > best_accuracy and f1_score_val > best_f1 and log_loss_val < best_log_loss:
            best_accuracy = accuracy_val
            best_f1 = f1_score_val
            best_log_loss = log_loss_val
            logistic_regression_best_clf = logistic_clf
            
            opt_iter = i
            opt_C = j

            arg_iter = cont_i
            arg_C = cont_j
        
        cont_j += 1

Comparing all those different models made us realize how robust each of them can be, but also how one can outperform the other depending on various factors like the problem itself, the time to train and the hyperparameters, for example.


Generally speaking, all of them performed well within our task, achieving a reasonably good accuracy and F1 score. The logistic regression, our baseline model and the simplest of them, was already able to reach up to 90% accuracy. The neural netowrk model, beeing a much more complex model and the hot topic of machine learning field for a while now, could beat the former model, reaching up to 93% accuracy. If we were to use more neural networks specific libraries - tensorflow, for example - and we could get a little more creative about the network architecture, it is possible that this model could have reached an even better score. Now for the SVM model, we got outstanding perfomance records, reaching up to 97% accuracy, although the training time came to be an hindrance, taking even longer than the neural network model.


Talking about the out of sample estimate, it was also really interesting to observe that the SVM, our chosen model based on the accuracy and F1 score on validation set, probably did not overfitted the training set given that it's scores on the test set were even better than on sub-validation and validation sets, reaching up to 98% accuracy and 0.98 F1 score.


After all our work, it is safe to say that anyone can already use our model to correctly classify their own digits ;)
